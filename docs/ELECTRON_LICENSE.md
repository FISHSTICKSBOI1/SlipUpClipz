# Electron License Validation

SlipUpClipz Pro licenses are validated **online** against the website backend. The desktop app treats license keys as opaque identifiers and never embeds Stripe, Netlify, or signing secrets.

## Endpoint

Configured in `electron/licenseConfig.ts`:

| Environment | URL |
| --- | --- |
| Production (packaged) | `https://slipupclipz.com/.netlify/functions/validate-license` |
| Local development | `http://localhost:8888/.netlify/functions/validate-license` |

Overrides:

- `SLIPUPCLIP_VALIDATE_URL` — force a custom endpoint in any build
- `SLIPUPCLIP_DEV_VALIDATE_URL` — dev-only override when unpackaged

## Activation flow

1. User enters a key in Settings → License (UI unchanged).
2. Main process POSTs `{ licenseKey }` to the validation endpoint over HTTPS (HTTP allowed only for local dev).
3. On success, the app stores:
   - `key`
   - `tier`
   - `expiresAt`
   - `lastValidatedAt`
4. On failure, the user sees a friendly message (invalid key, inactive subscription, expired, network error, temporary server error).

## Offline grace period

**7 days** after the last successful online validation.

- Pro features remain available offline within this window.
- After 7 days without a successful revalidation, Pro is disabled until the app can reach the server again.
- Local storage alone never permanently unlocks Pro.

## Revalidation frequency

- On activation (always online)
- On app startup (when a stored Pro key exists)
- Periodically at most **once every 12 hours**
- Not on page changes, soundboard actions, or other hot paths

## Development demo keys

Unpackaged development builds may still activate **legacy checksum demo keys** generated by `npm run license:demo` if server validation fails.

- Demo secret lives only in `electron/licenseDevDemo.ts`
- **Packaged production builds never accept demo keys**
- Paid Stripe-issued keys always require server validation

## Local stored fields

```typescript
{
  activated: boolean
  tier: 'free' | 'pro'
  key?: string
  activatedAt?: number
  expiresAt?: string | null
  lastValidatedAt?: number
}
```

No Stripe customer IDs, payment IDs, or webhook data are stored locally.

## Tests

```bash
npm run test:license
```

Integration scenarios (valid purchase, refunds, disputes, timeouts) should be verified against Stripe Test Mode before release.
